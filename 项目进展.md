## Reverted Experiment

An intro-video entry flow was implemented after commit 3cb8ec1 and then intentionally reverted as requested.

The reverted experiment included:

- Showing the existing loader first.
- Showing the Blue moment video after the loader.
- Starting background music after the first video click.
- Skipping the video on a second click.
- Attempting to start background music when the video ended.

The experiment was committed as 39c007d feat: add intro video before background music, but it is no longer part of the current repository state. The following changes were removed by restoring to 3cb8ec1:

- public/blue-moment.webm
- The loader completion postMessage hook in public/loading/script.js
- The intro-video state and UI changes in src/App.tsx

Important browser constraint: browsers generally block audible autoplay without a user gesture. A muted video can autoplay, but background music cannot be reliably started at the natural end of that video unless the user has already interacted with the page.

## Recommended Next Steps

1. Continue visual and interaction refinement from commit 3cb8ec1.

3. If an intro video is revisited, decide whether the first click should start music only or both start music and skip the video.
4. Test autoplay behavior separately on Chrome desktop, Safari desktop, and iOS Safari.
